# 証明

数学的な主張が正しいことを確かめるためには、多くの場合**証明**を行う。証明を適切に書くことで、それを読んだほかの人も主張の正しさを確かめることができる。

ここでは、証明とは何か、証明をするためには何をすればよいかを説明し、典型的な証明の手法をいくつか紹介する。

## 定理

ここでは、証明されて正しいことが明らかになった命題のことを**定理**と呼ぶことにする。

定理は、多くの場合**仮定**と**結論**からなる。次の例は、仮定と結論がわかりやすい例である：

!!! example "例"
    $n$ を整数とする。このとき、$n(n+1)$ は偶数である。

この例では、「$n$を整数とする」の部分が仮定であり、「$n(n+1)$ は偶数である」の部分が結論であると考えるのが普通であろう。

もう少し仮定や結論が複雑な場合も見てみよう。

!!! example "例"
    $f(x) \in \mathbb{Z}[x]$ を整数係数の多項式であって、任意の $a \in \mathbb{Z}$ に対して $f(a)$ が平方数であるものとする。このとき、ある整数係数の多項式 $g(x) \in \mathbb{Z}[x]$ が存在して、$f(x) = (g(x))^2$ が成り立つ。

(これはたぶん真の命題だと思うが、ちゃんと証明したことない。いい例が思い浮かんだら変える) この例では、「$f(x) \in \mathbb{Z}[x]$ を整数係数の多項式であって、任意の $a \in \mathbb{Z}$ に対して $f(a)$ が平方数であるものとする」までが仮定で、「ある整数係数の多項式 $g(x) \in \mathbb{Z}[x]$ が存在して、$f(x) = (g(x))^2$ が成り立つ」が結論である。

定理の内容を、しばしば定理の**ステートメント**と表現する。定理のステートメントが長くなってくると、仮定や結論が何であるかを把握するのが大変になることがある。結論が出てくる合図になりがちな表現として、「このとき」や英語の「Then」などがあるので、覚えておくとよいかもしれない。

- ここでいう「仮定」と「結論」はあまり数学的に厳密に意味があるつもりで分けているわけではない。
- 「任意の…に対して」のような部分と、「…ならば」の部分は仮定っぽい感じがする。
- 「$p$のとき、$q$ならば$r$」は、
    - 「$p$」が仮定で「$q$ならば$r$」が結論？
    - 「$p$」と「$q$」が仮定で「$r$」が結論？
- 厳密な話はともかく、日々の数学活動においては若干あいまいなところがある気がする。
- 「ある $x\in \mathbb{R}$ が存在して、$x^2 - 1 = 0$」は仮定が特にない定理？
    - こういうのは真面目に厳密にやろうとすると紛糾するのでよくないかもしれません

### 定理の呼び方について
- 数学書や論文では、「証明されて正しいことが明らかになった数学的主張」を表すための単語がいくつかある。使い分けは状況や著者の気分によって決まる。
    - 命題：もっとも汎用的に使うことができる。
        - このページでは偽の命題も命題と呼んでいるが、ふつうの数学の文章では、証明されたものだけを命題と呼び、偽の命題や、証明できるかわかっていない命題は命題とは呼ばないことに注意。
    - 定理：示せたら嬉しいものや、有名なもの、示すのが大変なものに対して使うことが多い気がする。
    - 補題：ほかの命題や定理を示すための準備となるものに対して使う。
    - 系：命題や定理からただちに導かれるものに対して使う。
    - 主張：主に証明中で、その命題や定理を示すための準備段階となるものに対して使う。
- 正しいかどうか(すなわち、証明できるかどうか)わかっていない数学的主張のことは、「予想」と呼ぶ。


## 証明

定理を証明するためには、(多くの場合)仮定から結論を導く必要がある。仮定と、すでに正しいとわかっていることを組み合わせて、主張を順々に示していき、最終的に結論が正しいということまで分かれば証明が完了したことになる。

(ここでLeanっぽく証明が進む図を書こうと思ったが、かえってわかりにくくなりそうなのでやめる。その代わり、パターンごとに典型的な証明の手法をいくつか紹介する)

## 証明の手法
以下のような状況でどうするかを書きたい気がするが、すべてを書くわけにはいかないかも。

- 「または」が仮定にあるとき
- 「または」を示すとき
- 「ならば」が仮定にあるとき
- 「ならば」を示すとき
- 「任意の」が仮定にあるとき
- 「任意の」を示すとき
- 「ある」が仮定にあるとき
- 「ある」を示すとき

「ある」を示すためには、大きく二つの方法がある：

- 条件を成り立たせる元を具体的に提示する。
- 抽象的に存在を保証する定理を利用する。

次の例は、具体的な元が提示しやすい例である：

!!! example "例"
    ある $x \in \mathbb{R}$ が存在して、$x^3 + x = 2$ が成り立つ。

**証明** $x=1$ とおくと、$x^3 + x = 1^3 + 1 = 2$ である。$\square$

これで証明が終わりである。このように、「ある」を示すためには、条件を満たす元が少なくとも一つあることを言えばよいので、具体的に例が出せればそれで十分である。

次の例は、具体的な元が提示しづらい例である：

!!! example "例"
    ある $x \in \mathbb{R}$ が存在して、$x^3 + x = 3$ が成り立つ。

**証明** 関数 $f(x) = x^3 + x - 3$ を考える。$f$ は $\mathbb{R}$ 上の連続関数である。$f(0)=-3$, $f(2) = 8+2-3 = 7$ である。したがって、中間値の定理より、ある $x \in (0,2)$ が存在して、$f(x) = 0$ が成り立つ。$\square$

中間値の定理は、ざっくりいうと、連続関数の符号が変わるのであれば、途中で必ず値がゼロになる点がある、という定理である。この定理は元の存在のみを主張しており、具体的にそれがいくつであるかに関する情報は保証していない。
抽象的に存在を保証する定理は便利であるが、それが結局どういう値なのかはわからないことも多い。

この例において、具体的な元を提示する方法で証明をしてみると、次のようになる：

**証明** $x = \sqrt[3]{\frac{27 + \sqrt{741}}{18}} + \sqrt[3]{\frac{27 - \sqrt{741}}{18}}$ とおくと、$x^3 + x = \cdots = 3$ である。

この二つを比較したとき、証明としてどちらがよいかは微妙なところかもしれない。具体的な $x$ の値が必要ないのであれば、抽象的な証明のほうが省エネでよいような気がするが、どうだろうか？
